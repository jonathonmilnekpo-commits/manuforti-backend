### API Endpoints Reference

```
POST /api/orders
  → Creates order, sends confirmation email, notifies Aiden
  → Returns: { order: { reference, status, payment_status, sla_deadline } }

GET /api/orders/:reference
  → Full order details (authenticated)

GET /api/orders/:reference/status
  → Public status check (minimal info)

POST /api/payments/stripe/create
  → Creates Stripe PaymentIntent
  → Returns: { client_secret, publishable_key }

POST /api/payments/vipps/create
  → Creates Vipps ePayment
  → Returns: { redirect_url, reference }

POST /api/payments/invoice/create
  → Activates order with invoice terms
  → Sends invoice email

POST /api/webhooks/stripe
  → Handles payment_intent.succeeded
  → Updates order status, notifies Aiden

POST /api/webhooks/vipps
  → Handles Vipps payment callbacks

GET /api/admin/orders
  → List all orders (requires API key)

GET /api/admin/orders/pending
  → Orders ready for processing

POST /api/admin/orders/:ref/activate
  → Manual activation (for invoice payments)
```

### Database Schema

```sql
orders (
  id SERIAL PRIMARY KEY,
  order_reference VARCHAR(20) UNIQUE NOT NULL,
  tier VARCHAR(20) NOT NULL,  -- Standard/Premium/Enterprise
  price INTEGER NOT NULL,
  payment_method VARCHAR(20) NOT NULL,  -- Vipps/Stripe/Invoice
  payment_status VARCHAR(20) DEFAULT 'pending',
  payment_id VARCHAR(255),
  
  -- Customer
  customer_name, customer_email, customer_phone, customer_title,
  
  -- Company
  company_name, industry, country, company_number, company_website,
  
  -- Supplier
  supplier_name, supplier_website, supplier_country, analysis_context,
  
  -- Status & SLA
  order_status VARCHAR(20) DEFAULT 'received',
  sla_hours INTEGER NOT NULL,
  sla_deadline TIMESTAMP NOT NULL,
  
  -- Timestamps
  created_at, paid_at, delivered_at
)
```

## Key Decisions Made

1. **Backend Stack:** Node.js/Express over Python/FastAPI — faster to deploy, better Stripe SDK support

2. **Database:** PostgreSQL — reliable, free tiers available on Railway/Render, excellent Node support

3. **Payment Flow:**

   - Vipps: ePayment API with redirect flow (Norwegian market)
   - Invoice: Immediate activation, 14-day terms, manual follow-up

5. **Security:**
   - Helmet.js for headers
   - Express validation for input sanitization
   - API key auth for admin endpoints
   - Webhook signature verification for Stripe
