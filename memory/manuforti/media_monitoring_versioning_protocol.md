# Media Monitoring Report Versioning Protocol

**Date:** April 1, 2026
**Status:** CRITICAL PROCESS LOCK

## Core Principle

**NEVER overwrite or lose previous report versions.** Each new version must build incrementally on prior work, maintaining:
- Previous analysis and findings
- Historical trend data
- Formatting and brand standards
- Client-specific customizations

## Version Progression Rules

### v1 → v2 → v3 → vn
- **v1:** Baseline report (initial coverage)
- **v2:** Enhanced with new methodology elements (add to v1, don't replace)
- **v3:** Full enterprise specification (add to v2, don't replace)
- **v4+:** Continuous build with new data/analysis layers

### File Naming Convention
```
[Company]_Media_Monitoring_v[VERSION]_[Month]_[Year].docx

Examples:
- Statkraft_Media_Monitoring_v1_March_2026.docx
- Statkraft_Media_Monitoring_v2_April_2026.docx
- Statkraft_Media_Monitoring_v3_April_2026.docx (Enterprise tier)
```

## Enterprise Tier Specification (v3 Requirements)

When running Enterprise level, include ALL of the following:

### 1. Report Frequency
- **Weekly reports** (not monthly)
- **Real-time alerts** for critical issues
- **Quarterly strategic reviews**

### 2. Monitoring Scope
- **All 4 social platforms:** X/Twitter, LinkedIn, Facebook, YouTube
- **Custom dashboards** (if implemented)
- **Unlimited suppliers** (within reason)
- **API access** for client integrations
- **WhatsApp alerts** for critical issues

### 3. Enhanced Search Protocol v2.0 (FULL IMPLEMENTATION)

#### Asset/Project-Level Monitoring
- Search company name + every known asset/project name
- Search company name + every operating country
- Track project lifecycle (announcement → construction → operation)

### 4. Report Sections (Enterprise Tier)

#### Section 1: Executive Dashboard
- Real-time risk score (0-100)
- Alert summary (critical items requiring immediate attention)
- Sentiment trajectory (week-over-week)
- Key metrics at a glance

#### Section 2: Critical Findings
- **URGENT:** Items requiring immediate client notification
- **HIGH:** Significant developments (legal, regulatory, ESG)
- **MEDIUM:** Notable mentions (market, competitive)
- **MONITOR:** Early signals (social media trends)

#### Section 3: Trend Analysis
- **30-day sentiment chart** (with annotations)
- **Volume analysis** (article count trends)
- **Source breakdown** (traditional vs. social)
- **Geographic distribution** (home market vs. operating countries)

#### Section 4: Traditional Media Coverage
- Comprehensive article table
- Sentiment by source type
- Key headlines analysis
- Editorial tone assessment

#### Section 5: Social Media Monitoring
- Platform-by-platform breakdown
- High-engagement posts
- Influencer mentions
- Hashtag tracking
- Sentiment by platform

#### Section 6: ESG & Regulatory
- Environmental controversies
- Social/license to operate issues
- Governance/legal proceedings
- Regulatory compliance status
- Indigenous rights monitoring

#### Section 7: Competitive Intelligence
- Competitor mentions
- Market positioning
- Industry trends affecting client
- Comparative sentiment analysis

#### Section 8: Risk Assessment
- Legal risk matrix
- Reputational risk factors
- Operational risk indicators
- Strategic recommendations

#### Section 9: Gap Analysis & Limitations
- What may have been missed
- Data access limitations
- Suggested follow-up searches
- Methodology improvements for next report

#### Section 10: Strategic Recommendations
- Immediate actions (next 7 days)
- Short-term priorities (next 30 days)
- Long-term strategic considerations
- Communication strategy suggestions

### 5. Quality Standards (Enterprise)

Before delivery, verify:
- [ ] All previous report findings incorporated
- [ ] No loss of historical data/trends
- [ ] Cover page with current date and version number
- [ ] Executive Summary on page 2
- [ ] Critical findings flagged with urgency indicators
- [ ] Sentiment trend charts generated
- [ ] All social platforms covered
- [ ] ESG section complete
- [ ] Gap analysis included
- [ ] Strategic recommendations actionable
- [ ] No placeholder text
- [ ] Professional formatting maintained
- [ ] File size appropriate (>200 KB for enterprise)

### 6. Version Control

**Each report must include:**
- Version number prominently displayed
- Previous version reference
- What's new in this version
- Continuity notes ("Building on v2 findings...")

**File naming:**
```
Statkraft_Media_Monitoring_v3_April_2026_Enterprise.docx
```

### 7. Handoff Protocol

When passing to next agent/session:
1. Reference previous version explicitly
2. List what was added vs. what was already there
3. Flag any formatting or content concerns
4. Confirm no data loss occurred

## Critical Reminder

**v3 is NOT a replacement for v2.**
**v3 is v2 + additional enterprise features + new data.**

If in doubt: **APPEND, don't REPLACE.**
