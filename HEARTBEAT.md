# HEARTBEAT.md — Aiden Memory & Context Sync

## Every 30 Minutes (07:00-23:00 Oslo Time)

### 1. Cross-Channel Context Sync
- Read memory/YYYY-MM-DD.md for today and yesterday
- Check for any new inputs from Telegram, WhatsApp, or other channels
- Consolidate scattered context into working memory

### 2. MEMORY.md Maintenance
- Review recent daily memory files (last 3 days)
- Identify significant decisions, learnings, or course corrections
- Update MEMORY.md with distilled key points
- Remove outdated information (keep it curated, not a dump)

### 3. Agent Coordination Check
- Check if Venture/Researcher/Validator agents ran and logged output
- Sync any agent decisions into main context
- Flag blockers or items needing Jonathon input

### 4. Daily Briefing Prep (if 07:00 or 19:00)
- Prepare mission statement briefing
- Check calendar for upcoming events
- Surface any urgent items

## Memory Update Rules

**ALWAYS update MEMORY.md when:**
- Pricing or product structure changes (like Category Strategy €3,999 lock)
- New blockers or HARD GATEs established/lifted
- Strategic decisions (SVP status, venture direction)
- New skills or capabilities added
- Significant client/prospect interactions
- Personal context (family, health, career moves)

**NEVER add to MEMORY.md:**
- Routine task completions
- Minor bug fixes
- Transient thoughts
- Duplicate information

## Cross-Channel Input Consolidation

When input arrives from any channel:
1. Log to memory/YYYY-MM-DD.md immediately
2. Flag if significant for MEMORY.md
3. Acknowledge receipt in originating channel
4. Surface in next terminal session if action needed

## Session Start Protocol (Terminal)

On every terminal session start:
1. Read SOUL.md, USER.md, MEMORY.md
2. Read memory/YYYY-MM-DD.md (today + yesterday)
3. Check for any pending inputs from other channels
4. Summarize context for Jonathon if significant gap detected
